# drizzle/

## Purpose
SQL migration files generated by drizzle-kit. Applied by `scripts/migrate.mjs` at container startup.

## Contents
- `0000_*.sql` ‚Äî Initial schema: users, topics, tasks, task_completions, Auth.js tables
- `0001_*.sql` ‚Äî Wishlist, achievements, push subscriptions, API keys, recurring tasks
- `0002_*.sql` ‚Äî OAuth account linking (linking_requests table)
- `0013_*.sql` ‚Äî notification_channels table (multi-channel notification system)
- `0014_*.sql` ‚Äî Two-factor authentication: `users.totp_secret` + `users.totp_enabled_at`, `sessions.totp_verified_at`, `totp_backup_codes` table
- `0015_passkeys.sql` ‚Äî WebAuthn / Passkeys: new `authenticators` table (Auth.js-compatible) + `RENAME COLUMN sessions.totp_verified_at ‚Üí second_factor_verified_at` (the gate is now method-agnostic)
- `0016_melted_black_cat.sql` ‚Äî Energy redesign: new `energy_checkins` table (history of every check-in, multiple per day allowed) with index `(user_id, date)` + new `topics.default_energy_level` column for topic-default-energy inheritance
- `0017_hard_boomer.sql` ‚Äî Sequential topics: new `topics.sequential` boolean column (default false) ‚Äî opt-in flag that restricts daily quest selection in a topic to the first still-open task
- `0018_smiling_lester.sql` ‚Äî iCal calendar feed: `users.calendar_feed_token_hash` (text, unique) + `users.calendar_feed_token_created_at` (timestamptz). Mirrors the `api_keys` token pattern ‚Äî plaintext shown once at creation, only SHA-256 hash persisted. Exactly one active feed token per user; rotation overwrites the hash and invalidates the previous URL immediately.
- `0019_low_mattie_franklin.sql` ‚Äî Due-today reminder: `users.due_today_reminder_enabled` boolean column (default false)
- `0020_*.sql` ‚Äî Streak Shield: `users.streak_shield_used_month` text column (nullable)
- `0021_boring_starjammers.sql` ‚Äî Onboarding wizard: `users.onboarding_completed` boolean column (default false) + backfill UPDATE setting all existing users to true
- `meta/_journal.json` ‚Äî Migration manifest: name, hash, timestamp for each migration
- `meta/*_snapshot.json` ‚Äî Drizzle schema snapshots (auto-generated, do not edit)

> Source of truth for the schema is `lib/db/schema.ts`, not the `.sql` files. The `.sql` files are deliberately excluded from agent context via `.claudeignore` (`drizzle/*.sql`) ‚Äî read them only when debugging an applied migration.

## Patterns
- Never edit `.sql` files manually after they've been applied to any environment
- New migrations: `npx drizzle-kit generate` ‚Üí review the generated SQL ‚Üí commit
- `meta/` files are auto-generated by drizzle-kit ‚Äî never edit manually
- Migration tracking lives in `drizzle.__drizzle_migrations` table (not `public` schema)
