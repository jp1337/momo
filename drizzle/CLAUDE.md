# drizzle/

## Purpose
SQL migration files generated by drizzle-kit. Applied by `scripts/migrate.mjs` at container startup.

## Contents
- `0000_*.sql` — Initial schema: users, topics, tasks, task_completions, Auth.js tables
- `0001_*.sql` — Wishlist, achievements, push subscriptions, API keys, recurring tasks
- `0002_*.sql` — OAuth account linking (linking_requests table)
- `0013_*.sql` — notification_channels table (multi-channel notification system)
- `0014_*.sql` — Two-factor authentication: `users.totp_secret` + `users.totp_enabled_at`, `sessions.totp_verified_at`, `totp_backup_codes` table
- `0015_passkeys.sql` — WebAuthn / Passkeys: new `authenticators` table (Auth.js-compatible) + `RENAME COLUMN sessions.totp_verified_at → second_factor_verified_at` (the gate is now method-agnostic)
- `0016_melted_black_cat.sql` — Energy redesign: new `energy_checkins` table (history of every check-in, multiple per day allowed) with index `(user_id, date)` + new `topics.default_energy_level` column for topic-default-energy inheritance
- `0017_hard_boomer.sql` — Sequential topics: new `topics.sequential` boolean column (default false) — opt-in flag that restricts daily quest selection in a topic to the first still-open task
- `0018_smiling_lester.sql` — iCal calendar feed: `users.calendar_feed_token_hash` (text, unique) + `users.calendar_feed_token_created_at` (timestamptz). Mirrors the `api_keys` token pattern — plaintext shown once at creation, only SHA-256 hash persisted. Exactly one active feed token per user; rotation overwrites the hash and invalidates the previous URL immediately.
- `0019_low_mattie_franklin.sql` — Due-today reminder: `users.due_today_reminder_enabled` boolean column (default false)
- `0020_*.sql` — Streak Shield: `users.streak_shield_used_month` text column (nullable)
- `0021_boring_starjammers.sql` — Onboarding wizard: `users.onboarding_completed` boolean column (default false) + backfill UPDATE setting all existing users to true
- `0022_closed_iron_lad.sql` — Notification history: new `notification_log` table (id, user_id, channel, title, body, status, error, sent_at) with index `(user_id, sent_at)` for the Settings history query and 30-day cleanup
- `0023_first_layla_miller.sql` — Morning Briefing: `users.morning_briefing_enabled` boolean (default false) + `users.morning_briefing_time` time (default '08:00') for the daily digest feature
- `0024_sudden_hydra.sql` — Vacation mode: `tasks.paused_at` date (nullable) + `tasks.paused_until` date (nullable) for per-task pause state, `users.vacation_end_date` date (nullable) for the global vacation toggle
- `meta/_journal.json` — Migration manifest: name, hash, timestamp for each migration
- `meta/*_snapshot.json` — Drizzle schema snapshots (auto-generated, do not edit)

> Source of truth for the schema is `lib/db/schema.ts`, not the `.sql` files. The `.sql` files are deliberately excluded from agent context via `.claudeignore` (`drizzle/*.sql`) — read them only when debugging an applied migration.

## Patterns
- Never edit `.sql` files manually after they've been applied to any environment
- New migrations: `npx drizzle-kit generate` → review the generated SQL → commit
- `meta/` files are auto-generated by drizzle-kit — never edit manually
- Migration tracking lives in `drizzle.__drizzle_migrations` table (not `public` schema)
